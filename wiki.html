FTheme is an open-source Flex 4 look and feel customization library. It provides a special Flex theme that changes components appearance dynamically.

What is the difference between FTheme and Spark theme?

  • FTheme allows to use unlimited amount of actual looks
  • Looks will work for future versions of your app and Flex SDK too
  • There will be a great amount of nice looks because it's fairly easy to create a look

What if I just want to change some backgrounds?

That's the way most looks are created. Here is a sample look JapaneseTatoo, note that fills reference bitmaps from ZIP archive:

So here are the simple look creation steps:

  1. Create MyLook.txt file and specify some fills or other styles
  2. If you use bitmap fills, place bitmaps with corresponding names in ZIP archive MyLook.zip. Note: do not apply compression to the archive, use STORE mode. We cannot yet decompress archives but we can read uncompressed ones.
  3. Upload look to the gallery

    OR

    Save it in near your app SWF and add it to your lookLinks.xml

How to manage looks that are available for the application?

Controller options

FThemeController constructor accepts one optional parameter options:FThemeOptions. It provides the following properties:

  • lookLinksXMLURL:String default URL for the file with look links XML file. Setting it e.g. to myLookLinks.xml will cause manager to load and use that file.
  • lookLinkNames:Array of String array of look names. Look files are loaded from LookName.txt and LookName.zip URLs.
  • lookLinksURLBase:String is used when lookLinkNames are specified and is added to look file URLs e.g. if lookLinkNames is ["Plastic", "Sky"/code> and lookLinksURLBase is "/app/looks/" then look files are loaded from "/app/looks/Plastic.txt" and "/app/looks/Sky.txt".
  • showDefaultLook is true by default and allows user to select default look in LookSelector UI component.

Specifying controller options via FlashVars

If options were not passed in controller constructor FlashVars are checked. The same options can be specified there:

<script type="text/javascript">
var flashvars = 
{
	lookLinksXMLURL: "/swf/lookLinks.xml",
	// lookLinkNames: "Plastic,Sky,Red",
	// lookLinksURLBase: "/some/dir/",
	// showDefaultLook: "false"
};
swfobject.embedSWF("/swf/app.swf", "swfContainer", "100%", "450", "10.1", 
	"/playerProductInstall.swf", flashvars);
</script>
<div id="swfContainer">

Default behavior - load lookLinks.xml

If controller options were not specified and FlashVars are empty then FTheme tries to load XML file lookLinks.xml and expects to get this:

<lookLinks>
	<lookLink name="Plastic"/>
	<lookLink name="Yakuza"/>
	<lookLink name="Red"/>
	...
</lookLinks>

<lookLink> node can also have the following attributes:

  • txtFileURL - URL to look txt file. If not specified Name.txt is used
  • zipFileURL - URL to look zip file. If not specified Name.zip is used if look has bitmap fills.

How does FTheme work?

It works nearly the same way as default spark.swc or wireframe.swc via defining skin for each component like Button, CheckBox and so on. The key difference is that FTheme does not define the actual pixels, they are drawn by fills and strokes that are generated at runtime.

As you see FTheme skin uses styles like buttonUpFill that is mx.graphics.IFill instance and buttonStroke that is mx.graphics.IStroke. That's what FTheme does - sets those styles to .global CSS style declatation inside Flex StyleManager.

What properties and values are supported in look?

Here is the list of currently supported look properties:

  • alternatingItemColors Array of colors Corresponds to Flex SDK "alternatingItemColors" style for list items backgrounds.
  • applicationFill Fill Application background fill.
  • borderStroke Stroke General stroke that is used in MX components borders and in other components that do not have implicit borders (like inputs that have "inputStroke").
  • borderStroke Stroke General border. Also sets "borderColor" and "borderAlpha" Flex SDK styles.
  • buttonCornerRadius Number Various buttons border corner radius.
  • buttonDisabledFill Fill Various buttons disabled fill.
  • buttonDownFill Fill Various buttons down fill.
  • buttonMinHeight Number Button min height. Default is 21.
  • buttonMinWidth Number Button min width. Default is 21.
  • buttonOverFill Fill Various buttons over fill.
  • buttonSelectedFill Fill Various buttons selected fill.
  • buttonStroke Stroke Various buttons border stroke.
  • buttonUpFill Fill Various buttons up fill.
  • checkBoxSize Number Check box width and height. Default is 13.
  • color null Text color in the application.
  • contentCornerRadius Number Content areas border corner radius.
  • contentFill Fill Content areas background fill e.g. list background fill.
  • contentStroke Stroke Content areas border stroke.
  • controlbarColor Color Controlbar text color.
  • controlbarFill Fill Controlbar is a component introduced by FTheme is much like mx:ControlBar but it can be placed everywhere in the application. Background fill.
  • controlbarStroke Stroke Controlbar top border stroke.
  • errorColor Color Error text color.
  • hScrollBarThumbDownFill Fill
  • hScrollBarThumbOverFill Fill
  • hScrollBarThumbUpFill Fill
  • hScrollBarTrackFill Fill
  • headerColors Array of colors Array of 2 colors for mx:DateChooser header fill.
  • headerFill Fill Data grid and some other components header fill.
  • highlightStroke Stroke Stroke that goes right next to the common interactive component border. Default is "0xFFFFFF alpha 0.2"/span>
  • inputCornerRadius Number Various inputs border corner radius.
  • inputFill Fill Various inputs background fill.
  • inputStroke Stroke Various inputs border stroke.
  • panelCornerRadius Number Panel corner radius.
  • panelFill Fill Panel content area background fill.
  • panelStroke Stroke Panel border stroke.
  • progressBarFill Fill Progress bar fill.
  • progressBarHeight Number Progress bar height.
  • progressBarStroke Stroke Progress bar border stroke.
  • progressBarTrackFill Fill Progress bar track fill.
  • radioButtonDotSize Number Radio button dot size. Default is 5
  • radioButtonSize Number Radio button width and height. Default is 13
  • rollOverColor Color Background color for hovered item renderers.
  • scrollBarButtonSize Number Defailt is 17.
  • scrollBarCornerRadius Number Defailt is 0.
  • scrollBarSize Number Defailt is 14.
  • scrollBarStroke Stroke
  • scrollBarThumbMinSize Number Defailt is 17.
  • selectionColor Color Background color for selected item renderers.
  • sliderThumbSize Number Defailt is 11.
  • sliderTrackCornerRadius Number Defailt is 3.
  • sliderTrackFill Fill
  • sliderTrackSize Number Defailt is 6.
  • sliderTrackStroke Stroke
  • symbolColor Color
  • symbolFill Fill
  • titlebarColor Color Titlebar text color.
  • titlebarFill Fill Titlebar fill. Titlebar is the topmost area of panel with panel title inside.
  • titlebarStroke Stroke Titlebar bottom border stroke.
  • toolbarButtonDisabledFill Fill Toolbar button disabled background fill.
  • toolbarButtonDownFill Fill Toolbar button down background fill.
  • toolbarButtonOverFill Fill Toolbar button over background fill.
  • toolbarButtonUpFill Fill ToolbarButton is a component introduced by FTheme. Button up background fill.
  • toolbarColor Color Toolbar text color.
  • toolbarFill Fill Toolbar background fill. Toolbar is a component introduced by FTheme. Is usually placed right below the panel titlebar.
  • toolbarStroke Stroke Toolbar bottom border stroke.
  • vScrollBarThumbDownFill Fill
  • vScrollBarThumbOverFill Fill
  • vScrollBarThumbUpFill Fill
  • vScrollBarTrackFill Fill

Types

Number

Simple number text form is supported e.g. buttonCornerRadius: 4

Color

The following string color representations are supported:
  • 0xFFCC00
  • #FFCC00
  • 0x333
  • #333

Array of colors

List of colors separated by spaces:
  • 0xFFCC00 0xFF0000 0xCC5500
  • #333 #222
  • 0x333

Fill

Linear or solid color fill

  • 0xFFCC00
  • 0xFFCC00 alpha 0.5
  • 0xFFCC00 0xFF0000 rotation 45
  • 0xFFCC00 alpha 0.5 ratio 0.2 0xFF0000 alpha 0.85 spreadMethod reflect scaleX 150

Valid values of spreadMethod are pad, reflect and repeat.

Default rotation is 90 so that gradient draws vertically.

Radial fill

  • radial 0xFFCC00
  • radial 0xFFCC00 alpha 0.5
  • radial 0xFFCC00 0xFF0000 rotation 45 scaleX 100 scaleY 200 focalPointRatio -1

Valid values of spreadMethod are pad, reflect and repeat.

Default rotation is 90.

Valid values of focalPointRatio are from -1.0 to 1.0.

Bitmap fill

  • bitmap assetId
  • bitmap assetId fillMode clip horizontalAlign right verticalAlign bottom smooth true
  • bitmap assetId fillMode clip horizontalAlign right y -30
  • bitmap assetId fillMode scale maintainAspectRatio true

Before specifying bitmap fill ensure that you've got bitmaps with corresponding names in your ZIP archive:

Stroke

Only solid strokes with alpha and pixelHinting are supported..
  • 0xFFCC00
  • #333 alpha 0.5 pixelHinting true
